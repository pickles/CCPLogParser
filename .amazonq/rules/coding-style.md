# コーディングスタイルガイド

## 基本原則
- 要件を満たす最小限のコードのみを記述
- 既存のパターンとスタイルを維持
- 不要なコメントや冗長な実装を避ける

## JavaScript/React規約
- ESLintルールに従う
- 関数型プログラミングを優先
- 適切なエラーハンドリングを実装
- パフォーマンスを考慮した実装

## 必須チェック
- .js/.jsx ファイルを修正した場合は必ずESLintで問題がないか確認する
- コード修正後は `npm run lint` または `npx eslint <ファイル名>` を実行
- ESLintエラーがある場合は修正してからコミット

## ファイル構造
- 機能別にファイルを分離
- 再利用可能なユーティリティは`utils/`に配置
- コンポーネントは`components/`に配置
